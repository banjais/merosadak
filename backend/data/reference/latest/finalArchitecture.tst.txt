Legend / Notes

Frontend → CF Worker → Backend

CF Worker handles caching GET requests and routing to Firebase (primary) or Render (fallback).

Controllers

Handle HTTP requests.

Call services for data fetch, filter, and processing.

Worker

Periodically fetches data (traffic, roads, weather, POIs, boundaries).

Precomputes search cache (searchCache).

Generates OTPs and JWT tokens for users.

Saves everything to /cache.

Services

Interface with external APIs, Google Sheets, and local GeoJSON files.

Provide standardized data objects to controllers and worker.

Data Folder (/backend/data)

Master GeoJSON, POIs, and boundary files.

/cache folder auto-updated by worker.

/logs auto-updated by logs.ts.

Logging & Alerts

All events/errors logged via logs.ts.

Critical failures can trigger email alerts via emailAlerts.ts.
