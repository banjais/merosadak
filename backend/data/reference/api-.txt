How it works

Frontend calls /api/* endpoints.

Cloudflare Worker intercepts requests:

Adds authorization headers (secrets hidden).

Forwards request to primary service (Cloudflare backend).

If fails, retries with secondary service (Render backend).

Backend services:

Handle search, AI, driver/vehicle health, Nepal boundary.

Can fetch Google Sheet, traffic, weather, POIs using secrets.

Secrets (API keys, tokens, Firebase keys) never exposed to users.

Frontend receives processed data:

Map overlays, traffic, POIs, AI suggestions, road status, driver/vehicle logs.

✅ Notes

POIs, weather, and traffic are optionally proxied through Worker to hide keys.

Fallback ensures service continuity if one backend is down.

Environment variables and secrets are set in Cloudflare, Render, and GitHub.

Frontend only sees the final endpoint (https://sadaksathi.web.app) — users don’t know if Cloudflare or Render is used.
