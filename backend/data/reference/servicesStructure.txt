Other services using data.service.ts:
- dhm.service.ts → fetchDhm
- search.service.ts → fetchRoads + fetchPOIData
- voice.service.ts → TTS queue (optional travel summaries)
- radio.service.ts → static / mock radio stations

✅ Notes:

data.service.ts is the core: All live/mocked data flows through it.

Mocks are fallback only if API fails or USE_MOCK = true.

Road / master.geojson is your authoritative source for roads/highways.

Frontend calls backend services (poi.service.ts, traffic.service.ts, road.service.ts, etc.) – never directly to data.service.ts.

.env.local variables like DATA_PATH, API keys, and endpoints are read inside services.
